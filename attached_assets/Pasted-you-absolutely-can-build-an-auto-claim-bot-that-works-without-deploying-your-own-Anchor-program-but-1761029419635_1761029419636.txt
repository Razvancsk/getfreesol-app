you absolutely can build an auto-claim bot that works without deploying your own Anchor program, but with limitations.

Let’s break it down clearly so you know what’s possible, what’s not, and how to build it right 👇

⚙️ YES — you can do it with a bot only (no on-chain program)
🔹 What it means

Your “Auto-Claim” system can live entirely off-chain (backend bot + relayer)
and use existing Solana system instructions and the SPL Token program.

That means:

You don’t need to deploy a custom Anchor program.

You just send transactions directly to Solana’s Token Program (111111...) using the standard CloseAccount instruction.

Your bot can auto-detect empty accounts and close them for the user.

🧠 How It Works

Here’s how your off-chain Auto-Claim bot would function:

🧩 Step 1 — User Grants Access

You still need one of these setups:

User stays online → wallet signs each claim
(Not ideal for automation, but works immediately)

User gives authority once → bot can claim anytime

The user runs a one-time SetAuthority transaction.

Changes the closeAuthority of their token accounts to your bot’s PDA or relayer wallet.

After that, your backend can claim anytime without user online.

🔑 This is the key: the SPL Token Program requires the closeAuthority to sign any claim.
Without it, no bot or meta-tx can close the account.

🧩 Step 2 — Bot Watches the Blockchain

Your bot (Node.js or Rust) listens for:

New token accounts owned by that user

Empty accounts (balance = 0)

You can use:

@solana/web3.js
@solana/spl-token


and Solana RPC/WebSocket subscriptions to detect them.

🧩 Step 3 — Bot Sends Close Transaction

When it finds empty accounts:

import { closeAccount } from "@solana/spl-token";
import { Connection, sendAndConfirmTransaction, Transaction } from "@solana/web3.js";

const tx = new Transaction().add(
  closeAccount({
    source: accountToClose,
    destination: userWallet,      // rent refund destination
    authority: botWallet.publicKey,
  })
);

await sendAndConfirmTransaction(connection, tx, [botWallet]);
