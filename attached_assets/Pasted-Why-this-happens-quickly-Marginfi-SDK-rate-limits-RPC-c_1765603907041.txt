RPC calls

Account fetches

Bank & group data queries

If:

multiple users load the Earn page at once

the page refetches on every render

you don’t cache responses

➡️ the SDK throws 429 Too Many Requests

✅ Recommended Architecture (Best Practice)
1️⃣ Primary source: Marginfi SDK (live data)
2️⃣ Fallback source: Cached API / stored snapshot
3️⃣ Graceful UI degradation
1️⃣ Wrap Marginfi SDK calls with retry + timeout
async function withTimeout<T>(
  promise: Promise<T>,
  ms = 5000
): Promise<T> {
  const timeout = new Promise<never>((_, reject) =>
    setTimeout(() => reject(new Error("Timeout")), ms)
  )
  return Promise.race([promise, timeout])
}

async function fetchMarginfiData() {
  try {
    return await withTimeout(
      marginfiClient.getBanks(),
      4000
    )
  } catch (err: any) {
    if (err?.message?.includes("429")) {
      throw new Error("RATE_LIMIT")
    }
    throw err
  }
}

2️⃣ Add a cached fallback API (IMPORTANT)

Create a server-side cron / job that fetches Marginfi data periodically (e.g. every 1–5 minutes) and stores it.

Example:

Redis

Supabase

PostgreSQL

KV store (Vercel KV / Cloudflare KV)

// cron job
const banks = await marginfiClient.getBanks()

await redis.set(
  "marginfi:banks",
  JSON.stringify(banks),
  { EX: 60 } // cache for 60s
)

3️⃣ Fallback logic on the Earn page
async function loadEarnData() {
  try {
    return {
      source: "live",
      data: await fetchMarginfiData()
    }
  } catch (err: any) {
    if (err.message === "RATE_LIMIT") {
      const cached = await redis.get("marginfi:banks")
      if (cached) {
        return {
          source: "cached",
          data: JSON.parse(cached)
        }
      }
    }
    throw err
  }
}

4️⃣ UI: Show data even when SDK fails
{source === "cached" && (
  <div className="text-yellow-500 text-sm">
    Showing cached data (live data temporarily unavailable)
  </div>
)}


This avoids:

blank Earn pages

bad UX

5️⃣ Reduce SDK calls (very important)
❌ Don’t do this
useEffect(() => {
  marginfiClient.getBanks()
}, [])

✅ Do this

Fetch once

Cache globally

Share across components

const { data } = useSWR(
  "marginfi-banks",
  fetchMarginfiData,
  { refreshInterval: 60000 }
)

6️⃣ Extra protection (recommended)
Add exponential backoff
import pRetry from "p-retry"

const data = await pRetry(
  () => fetchMarginfiData(),
  { retries: 3 }
)
